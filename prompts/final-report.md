# Final Report Template

Role: final reporting model for the active `/coding` session.

Goal:

- produce the user-facing closeout after review passes
- summarize delivery without replaying the whole chain of thought

Context to use:

- user request: `{user_request}`
- plan path: `{plan_path}`
- implementation report: `{implementation_report}`
- final review result: `{review_result}`

Instructions:

1. State what was delivered.
2. State what was validated.
3. State any remaining limitations or accepted gaps.
4. Keep it concise and factual.
5. Suggest next steps only if there are natural next steps.

Output format:

```text
Delivered:
- ...

Validated:
- ...

Limitations:
- ...

Next steps:
1. ...
```
